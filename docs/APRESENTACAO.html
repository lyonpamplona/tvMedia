tvMedia Studio

Controle telas, midias, playlists, campanhas, DataSets, widgets e proof-of-play em um unico sistema FastAPI com painel web e player para TVs em tempo real.

FastAPI SQLAlchemy WebSocket PWA sem build 2FA TOTP Backups SQLite

Zona principal

Videos, imagens, HTML, campanhas e playlists resolvidas pelo CMS.

Widgets

RSS, clima, QR, stocks, menu board e DataSets.

BI

Proof-of-play, CSV e relatorios por e-mail.

Recursos do Sistema

Studio AdminEditor visual para midias, playlists, telas, zonas, overlays, agendas, campanhas, DataSets, BI, usuarios, auditoria e plataforma.
Player TVAplicacao fullscreen que consome `/api/display/{slug}`, conecta via WebSocket e atualiza quando a revisao muda.
OperacaoHealthcheck, alertas offline, comandos de tela, screenshot best-effort, backups e relatorios agendados.
SegurancaUsuarios, papeis, multiempresa, super admin, 2FA TOTP, tokens Bearer, rate-limit e validacao de segredos em producao.
Conteudo dinamicoDataSets manuais, CSV e JSON remoto, widgets de noticias, cotacoes, relogio, calendario, menu board e stocks.
BIProof-of-play com resumo, detalhes, exportacao CSV e envio por SMTP com anexo PDF textual.

Arquitetura

O backend FastAPI centraliza API, WebSocket, tarefas em background, persistencia e frontend estatico. O player e independente, publico por slug, e recebe um payload ja resolvido pelo servidor.

Admin Web -> REST autenticado -> FastAPI -> SQLAlchemy -> SQLite/PostgreSQL
Player TV -> GET /api/display/{slug}
Player TV -> WS /ws/display/{slug}
FastAPI lifespan -> backup_scheduler + offline_alert_scheduler + report_scheduler
CamadaResponsabilidade
BackendAPI REST, auth, modelos, regras de negocio, payload do player, BI, backups e WebSocket.
AdminPainel PWA sem build para configurar todo o CMS.
PlayerRenderizacao fullscreen das zonas, midias, widgets, comandos e eventos de exibicao.
InfraDocker Compose, `.env`, healthcheck, volume persistente e CI GitHub Actions.

Fluxos do Sistema

Conteudo ate a TV
Criar midia no admin.
Adicionar midia a playlist.
Vincular playlist a zona, agenda ou campanha.
Player busca payload e renderiza.
Tempo real
Player conecta no WebSocket da tela.
Admin altera conteudo.
Backend envia `reload`.
Player recarrega se a `revision` mudou.

Outros fluxos cobertos

  • Login, 2FA, sessao HMAC e API tokens.
  • Agendamento por fuso, prioridade e playlist padrao.
  • Campanhas com alvo por tela, grupo ou zona e limite por hora.
  • Proof-of-play, CSV, relatorios agendados e coleta seletiva.
  • Backups automaticos/manual e download administrativo.

API Tecnica

Rotas administrativas usam `Authorization: Bearer`. O Bearer pode ser token de sessao ou token pessoal `tvma_...`.

FamiliaRotas
Auth`/api/auth/login`, `/me`, `/change-password`, `/logout`, `/2fa/*`, `/api-tokens`
Midias`/api/media`, `/upload`, `/bulk`, `/bulk-tags`, `/import-url`, `/purge-unused`
Playlists`/api/playlists`, `/items`, `/reorder`, `/import`, `/export`, `/folders`
Telas`/api/screens`, `/publish`, `/layout-lock`, `/commands`, `/screenshot`, `/map`
Display publico`/api/display/{slug}`, `/api/display/{slug}/events`, `WS /ws/display/{slug}`
Dados`/api/datasets`, `/api/widgets/news`, `/rates`, `/stocks`, `/datasets/{id}`
Analytics`/api/analytics/proof-of-play/*`, `/reports`, `/screens/health`
Sistema`/api/system/backup`, `/api/system/backups`

Configuracao e Dependencias

DependenciasPython 3.12+, FastAPI, SQLAlchemy, Pydantic, Uvicorn, requests, python-multipart, Alembic, Pillow opcional, ffmpeg/ffprobe opcional.
AmbienteConfigure `.env`, use `ENVIRONMENT=production`, troque `SECRET_KEY`, troque `ADMIN_PASSWORD` e restrinja `CORS_ORIGINS`.
VariavelUso
`DATABASE_URL`SQLite por padrao ou PostgreSQL em producao.
`MEDIA_DIR`Pasta persistente das midias.
`FORCE_HTTPS`Redirect HTTPS atras de proxy confiavel.
`BACKUP_*`Backup automatico SQLite e retencao.
`SMTP_*`Alertas e relatorios por e-mail.
`REPORT_SCHEDULER_*`Agendador de BI.

Como Usar

Copie `.env.example` para `.env` e ajuste segredos.
Suba com `docker compose up --build -d`.
Acesse `/admin/` e faca login.
Crie midias e uma playlist.
Crie uma tela, configure a zona principal e copie a URL do player.
Abra `/player/?screen=SLUG` na TV em modo kiosk.
Use BI, campanhas, DataSets, 2FA e backups conforme a operacao crescer.
